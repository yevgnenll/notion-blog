# 블로그 포스팅 SEO 전략 가이드

구글의 [SEO Starter Guide](https://developers.google.com/search/docs/fundamentals/seo-starter-guide) 원칙을 기반으로 한 블로그 작성 체크리스트.

---

## 1. 사용자 중심의 유용하고 신뢰할 수 있는 콘텐츠 (Helpful, Reliable, People-first Content)

- 단순 결론("이렇게 해라")에 그치지 말고, **왜** 그래야 하는지 근거와 원리를 함께 설명
- 수치, 공식, 전공 지식, 공식 문서 인용 등으로 신뢰성 확보
- 독자가 핵심 개념을 스스로 이해할 수 있도록 깊이 있게 서술

## 2. 독자의 검색어 예측 (Expect Your Readers' Search Terms)

- 독자가 문제 상황에서 실제로 검색할 키워드를 글의 **자연스러운 문맥 속**에 배치
- 예: `filesort`, `ORDER BY 성능`, `디스크 I/O`, `실행 계획`, `sort_buffer_size`
- 키워드를 억지로 나열하지 않고 설명의 흐름 안에 녹여 사용

## 3. 가독성 높은 구조와 시맨틱 태그 활용 (Well-organized Text)

- 제목 계층 구조를 의미에 맞게 사용
  - `#` (H1): 포스트 제목 — 문서당 하나
  - `##` (H2): 주요 섹션
  - `###` (H3): 세부 항목
- 구글 크롤러와 스크린 리더가 글의 구조를 쉽게 파악할 수 있도록 구성
- 목록, 코드 블록, 표 등을 적극 활용해 스캔 가독성 향상

## 4. 고유하고 명확한 제목 (Influence Your Title Links)

- 제목에 핵심 주제 키워드를 포함하되 자연스럽게 작성
- 검색 결과에서 클릭을 유도할 수 있도록 구체적이고 매력적으로 구성
- 예: "데이터베이스 성능 최적화의 비밀: Filesort와 외부 정렬-병합 알고리즘의 수학적 이해"
  - 포함 요소: 문제 영역(DB 성능) + 핵심 개념(Filesort) + 차별점(수학적 원리)

## 5. 키워드 남용 방지 (Avoid Keyword Stuffing)

- 특정 단어를 의미 없이 반복 나열하지 않기
- 같은 개념을 다양한 관련 어휘로 표현해 자연스러운 언어 매칭 유도
  - 예: "정렬" → 외부 정렬, 런 생성, 병합 단계, sort_buffer_size, filesort 등으로 분산

---

## 작성 체크리스트

- [ ] 제목에 핵심 키워드 포함, 60자 이내
- [ ] 서론에서 독자의 문제 상황을 명확히 언급
- [ ] H2/H3 소제목에 검색 가능한 키워드 자연스럽게 포함
- [ ] 주장마다 근거(수치, 공식, 공식 문서, 인용) 첨부
- [ ] 동일 키워드 3회 이상 연속 반복 없음
- [ ] 결론/요약 섹션에서 핵심 내용 재정리
- [ ] 태그는 3~5개, 글의 핵심 주제와 직결된 단어로 선정
- [ ] 3000자 이상 작성 (롱폼 콘텐츠가 SEO에 유리)
